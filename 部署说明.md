# MinimalistPicks · 傻瓜部署 & 日常运营指南

> 零依赖、零构建、纯静态。不需要装 Node，不需要懂代码。会用浏览器和 GitHub 网页就够了。

---

## 一、3 分钟上线（GitHub Pages）

### 第 1 步：在 GitHub 新建一个仓库
1. 登录 [github.com](https://github.com)
2. 右上角 `+` → **New repository**
3. 仓库名建议直接用你的域名前缀，例如 `minimalistpicks`
4. 选 **Public**（公开），底下不勾任何选项
5. 点 **Create repository**

### 第 2 步：把这些文件全部上传
1. 仓库页面点 **uploading an existing file**（或拖文件区）
2. 把本文件夹里**所有文件和文件夹**拖进去（包括 `.nojekyll`、`assets/`、`posts/`、`tools/`）
3. 底部点 **Commit changes**

> ⚠️ Mac 用户：上传前确认 `.nojekyll` 没被隐藏。Finder 按 `⌘ + Shift + .` 显示隐藏文件。

### 第 3 步：打开 Pages 开关
1. 仓库 → **Settings** → 左侧 **Pages**
2. **Source** 选 `Deploy from a branch`
3. **Branch** 选 `main` + `/ (root)` → **Save**
4. 等 1–2 分钟，最上方出现网址：`https://你的用户名.github.io/minimalistpicks/`

**✅ 网站已上线！**

## 二、绑定自己的域名（可选）

1. 改根目录 **CNAME** 文件内容为你的域名（例如 `minimalistpicks.com`）
2. 在域名注册商面板加 DNS：
   - 4 条 A 记录：`185.199.108.153` / `185.199.109.153` / `185.199.110.153` / `185.199.111.153`
   - 1 条 CNAME：`www` → `你的用户名.github.io`
3. GitHub → Settings → Pages → **Custom domain** 填域名 → 勾 **Enforce HTTPS**

不绑定独立域名的话：删掉 `CNAME` 文件，把 `config.js` 里的 `siteUrl` 改成你的 GitHub Pages 网址。

## 三、🎯 发新文章（重点：现在改用 Markdown 文件）

每篇文章 = `posts/` 文件夹里的**一个 `.md` 文件**。整套流程只有 2 步：

### 第 1 步：在 `posts/` 文件夹新建一个 `.md` 文件

文件名就是文章网址 slug，**只能用小写英文字母、数字、短横线**。
例如 `posts/best-coffee-grinder-2026.md`

把下面这段复制进去当模板：

```markdown
---
title: 你的文章标题
date: 2026-05-15
score: 4.8
cover: https://images.unsplash.com/photo-XXXXX?w=1200
intro: 用户的痛点开场，一两句话。这段会出现在首页卡片 + 文章开头。
amazonKeyword: amazon 搜索关键词
metaDesc: SEO 描述，150 字以内。
tags: 标签1, 标签2
---

正文从这里开始，直接写就行。可以分很多段。

![这是图片的描述](https://images.unsplash.com/photo-XXXXX?w=1200)

## 二级标题

继续正文段落...

- 亮点一
- 亮点二
- 亮点三

> 引用一句话，会用衬线字体优雅显示。

可以**加粗**，可以*斜体*，可以加 [链接](https://example.com)。

![另一张图片](images/my-article/photo-2.jpg)

收尾段落。
```

### 第 2 步：在 `articles.js` 里加一行 slug

打开 `articles.js`，在数组里加一行你的文件名（不带 `.md` 后缀）：

```js
window.ARTICLES = [
  "bamboo-monitor-stand-review",
  "stonewashed-linen-throw-blanket",
  "braided-usb-c-cable-6ft",
  "best-coffee-grinder-2026"      // ← 你的新文章
];
```

注意：每一项加双引号，每行末尾加逗号（最后一项可加可不加）。

### 第 3 步：提交

把这两个文件（新 `.md` + 改过的 `articles.js`）一起提交到 GitHub，**30 秒后刷新网站，新文章就上线了**。

## 四、Markdown 速查表（一看就会）

| 你写 | 显示效果 |
|------|----------|
| `## 标题` | 二级标题（用得最多） |
| `### 小标题` | 三级标题 |
| `**加粗**` | **加粗** |
| `*斜体*` | *斜体* |
| `[文字](网址)` | 可点击的链接 |
| `![图片描述](图片URL)` | 显示一张图片（自带懒加载） |
| `- 第一项` | 项目符号列表（连续几行就成一个列表） |
| `1. 第一项` | 数字列表 |
| `> 引用文字` | 优雅的引用块（衬线字体） |
| `---` | 一条分隔线 |
| `` `代码` `` | 行内代码 |
| 空一行 | 分段 |

**写文章就像写微信文章一样自然 —— 不用想"标签"、"HTML"、"JS"，纯文字 + 上面这几个符号就够了。**

## 五、🖼️ 插图：两种方式

### 方式一：用网上的图（最简单）

去 [unsplash.com](https://unsplash.com) 找免费高质量图，右键图片 → "复制图像地址"，得到一个 https 链接，直接写在 markdown 里：

```markdown
![一杯手冲咖啡](https://images.unsplash.com/photo-XXXXX?w=1200)
```

**优点**：免费、不占空间、加载也快
**缺点**：图床哪天挂了你就没图了（Unsplash 多年很稳）

### 方式二：用自己的图（自主可控）

1. 把图片文件拖进 `posts/images/` 文件夹（建议按文章建子文件夹）
2. 在 markdown 里写**相对路径**：

```markdown
![我拍的手冲咖啡](images/coffee-grinder/photo-1.jpg)
```

**优点**：完全自主，不怕图床挂
**缺点**：占 GitHub 仓库空间，需要自己压图

**图片建议**：
- 宽度 1200–1600px 足够
- 单张控制在 200KB 以内（[tinypng.com](https://tinypng.com) 在线压缩）
- 用 `.webp` 格式体积最小（手机也都支持）

## 六、改全站设置：只动 `config.js`

- 想换站名 → `siteName`
- 想换 slogan → `tagline`
- 想换联系邮箱 → `contactEmail`
- 想加 Google Analytics → `analyticsId` 填 `G-XXXXXXXXXX`
- 想加 AdSense → `adsenseId` 填 `ca-pub-XXXXXXXXXXXXXXXX`
- 想换亚马逊联盟 tag → `amazonTag`
- 想关掉某个分享平台 → `enableShare` 改对应 `true`/`false`

法律页面 / 页脚 / 标题 / 邮箱链接 都会自动同步，不用逐页改。

## 七、更新站点地图（让 Google 知道新文章）

每次发完新文章做一次（10 秒）：

1. 浏览器打开 `你的网址/tools/update-sitemap.html`
2. 点 **"⬇️ 下载为 sitemap.xml"**
3. 把下载的文件，拖进 GitHub 仓库覆盖根目录的 `sitemap.xml`，提交即可

> 第一次部署后，记得去 [Google Search Console](https://search.google.com/search-console) 添加你的站点，提交一次 `sitemap.xml`。

## 十、常见问题

**Q：新文章怎么不显示？**
A：检查 3 点：
1. `.md` 文件确实在 `posts/` 文件夹里
2. `articles.js` 里加了对应的 slug（小写英文 + 短横线，要和文件名一致）
3. 强刷浏览器：Mac `⌘+Shift+R`，Win `Ctrl+F5`

**Q：图片不显示？**
A：
- 外链图：检查 URL 是不是直接图片地址（结尾 `.jpg`/`.png`/`.webp`），不是网页地址
- 本地图：路径写成 `images/my-folder/pic.jpg`（不要写 `/posts/images/...`，会错）

**Q：能不能不写 `intro` / `tags` / `score`？**
A：能。除了 `title`，其他字段都可选。但建议都填，对 SEO 和排序有用。

**Q：怎么换字体 / 颜色 / 暗色模式？**
A：改 `assets/style.css` 顶部 `:root` 里的 CSS 变量。暗色模式已自动跟随系统。

**Q：能不能放视频？**
A：可以。在 markdown 里直接写 HTML 也支持，比如：
```html
<video src="images/demo.mp4" controls style="width:100%;border-radius:10px;"></video>
```

**Q：可以预览效果再发吗？**
A：可以。把整个文件夹用 VS Code 打开，装一个 "Live Server" 插件右键 "Open with Live Server"，本地实时看效果。或者直接在 GitHub 的 main 分支以外建个 `draft` 分支预览。
